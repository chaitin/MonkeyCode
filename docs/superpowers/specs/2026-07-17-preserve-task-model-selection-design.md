# 启动任务模型选择保持设计

## 背景

侧边栏“启动任务”弹窗将模型、主机和镜像的默认值集中在 `setDefaultConfig` 中。弹窗打开期间，订阅、模型、主机或镜像数据更新会重建该回调并再次执行默认配置，覆盖用户手动选择的模型。

## 目标

- 弹窗打开后，在模型和订阅数据可用时初始化套餐推荐模型。
- 用户手动选择有效模型后，订阅、模型、主机或镜像刷新继续保留该选择。
- 用户选择的模型被删除或订阅权限失效后，清空选择并要求用户重新选择。
- 弹窗关闭后清除用户选择标记，使下一次打开可以重新初始化默认模型。
- 创建请求继续提交当前 `selectedModelId`，后端接口和持久化逻辑保持不变。

## 方案

在 `CreateDefaultTaskDialog` 中增加 `modelTouchedRef`，将模型初始化从主机和镜像默认配置中拆分：

1. `ModelSelect` 的变更回调先标记用户已操作，再更新 `selectedModelId`。
2. 模型初始化 effect 在弹窗打开且模型列表可用时调用 `selectPreferredTaskModel`，并通过纯函数解析当前选择状态。
3. 用户尚未操作时同步套餐推荐模型；用户选择仍有效时保持该选择；模型被删除或权限失效时清空选择。
4. 主机和镜像继续根据弹窗状态及数据更新初始化，但不再写入模型状态。
5. 弹窗关闭时重置 `modelTouchedRef` 和模型状态。

该方案沿用 Issue 任务弹窗的现有模式，兼容弹窗早于异步模型数据打开的场景，也能阻止 30 秒订阅刷新覆盖用户选择。

## 验证

- 行为测试执行生产状态函数，确认默认初始化、刷新保留和失效清空规则。
- 契约测试确认用户 touched 标记、状态函数接线和关闭重置逻辑。
- 契约测试确认创建请求提交 `selectedModelId`。
- 运行前端专项测试、`pnpm lint` 和 `pnpm run build:online`。
- SaaS 人工验收：选择非推荐模型并等待超过 30 秒，确认选择和最终任务模型保持一致。
