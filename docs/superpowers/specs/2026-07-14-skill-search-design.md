# 任务 Skills 检索功能设计

## 背景

普通用户在创建任务时通过 Skills 选择弹窗选择需要的 Skill。随着技能数量增加，仅依靠标签切换和滚动列表难以快速定位目标 Skill，对应 GitHub Issue #763。

## 范围

- 在共享的任务 Skills 选择弹窗中增加检索框。
- 任务输入、默认任务创建和项目开始开发三个入口统一生效。
- 输入关键词时即时过滤当前已加载的 Skills。
- 检索覆盖 Skill 名称、描述和标签。
- 检索忽略大小写，并忽略关键词首尾空白。
- 搜索结果与当前选中的标签取交集。
- 关闭弹窗后清空关键词，再次打开时显示当前标签下的完整列表。
- 没有匹配项时显示独立的空结果提示。

## 方案

三个任务入口继续通过现有 `v1SkillsList` API 获取 Skills，并将数据传给共享 `TaskSkillSelector`。选择器内部新增搜索关键词状态，在渲染阶段根据关键词和当前标签过滤 `skills`，无需新增 API 或修改调用方的数据结构。

搜索框放在弹窗标签栏上方，使用项目现有 `Input` 和搜索图标。各标签页渲染同一份关键词过滤结果；当前标签没有匹配项时显示“未找到匹配的 Skill”。

选择 Skill、切换标签和强制下发 Skill 的既有行为保持不变。搜索不会修改、排序或持久化 Skill 数据，也不会触发额外网络请求。关闭弹窗时清空内部搜索状态。

## 数据匹配

每个 Skill 的以下字段合并参与包含匹配：

- `name`
- `description`
- `tags`

关键词和候选字段统一去除大小写差异，关键词同时去除首尾空白。空字段按空字符串处理。搜索条件与标签条件同时满足时才展示 Skill。

## 文案

在 `taskWorkflow.skill` 中增加中英文文案：

- 搜索框占位文案
- 搜索无结果提示

## 验收标准

- 普通用户可在任务输入、默认任务创建和项目开始开发三个入口看到搜索框。
- 输入名称、描述或标签中的任意片段，可以找到对应 Skill。
- 英文字段检索不区分大小写。
- 搜索结果同时符合当前标签筛选条件。
- 清空关键词后恢复当前标签下的完整列表和原始顺序。
- 关闭并重新打开弹窗后，搜索关键词为空。
- 无匹配项时显示搜索空结果。
- 选择、取消选择、默认 Skill 和强制下发 Skill 行为保持不变。
- 弹窗在桌面端和移动端均可正常使用。
- `pnpm lint` 与 `pnpm run build:online` 通过。
