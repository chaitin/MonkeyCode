# Skills 检索功能设计

## 背景

团队 Skills 管理页会一次性加载全部 Skill。随着数量增加，用户需要通过检索快速定位目标 Skill，对应 GitHub Issue #763。

## 范围

- 在团队 Skills 管理页的列表上方增加检索框。
- 输入关键词时即时过滤当前已加载的 Skills。
- 检索覆盖 Skill 名称、描述、标签、分组名称、来源名称和 `SKILL.md` 路径。
- 检索忽略大小写，并忽略关键词首尾空白。
- 无关键词时保持接口返回的原始顺序。
- 有 Skills 但没有匹配项时显示独立的空结果提示。

## 方案

页面继续通过现有 `v1TeamsSkillsList` API 一次性获取 Skills。组件新增搜索关键词状态，在渲染阶段根据关键词过滤 `skills`，无需新增 API 或修改数据结构。

搜索框放在 `CardContent` 的列表上方，使用项目现有 `Input` 和搜索图标。加载期间禁用输入框；团队没有任何 Skill 时保留现有空状态；搜索无结果时显示“未找到匹配的 Skill”，避免与“暂无 Skills”混淆。

新增、编辑或删除 Skill 后，过滤结果直接基于最新 `skills` 状态重新计算。搜索不会修改、排序或持久化 Skill 数据。

## 数据匹配

每个 Skill 的以下字段合并参与包含匹配：

- `name`
- `description`
- `tags`
- `groups[].name`
- `sourceLabel`
- `skillMdPath`

关键词和候选字段统一转换为小写。空字段按空字符串处理。

## 文案

在 `managerSkills` 中增加中英文文案：

- 搜索框占位文案
- 搜索无结果标题
- 搜索无结果说明

## 验收标准

- 输入名称、描述、标签、分组、来源或路径中的任意片段，可以找到对应 Skill。
- 英文字段检索不区分大小写。
- 清空关键词后恢复完整列表和原始顺序。
- 无匹配项时显示搜索空结果，现有“暂无 Skills”状态保持不变。
- 新增、编辑和删除操作完成后，当前检索条件继续生效。
- 页面在桌面端和移动端均可正常使用。
- `pnpm lint` 与 `pnpm run build:online` 通过。
