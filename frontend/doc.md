# MonkeyCode AI 前端产品使用文档（功能说明版）

本文档基于当前 `frontend` 代码实现整理，按“功能项”逐条说明：

- 这是个什么功能
- 什么时候使用
- 对用户/团队有什么价值

适用对象：产品、售前、实施、测试、运营、研发同学。

---

## 1. 角色与产品主线

### 功能：普通用户控制台

- 功能定义：面向开发者的主工作区，包含任务、项目、Git 机器人、终端和文件管理能力。
- 使用时机：个人或小组成员要发起 AI 开发任务、查看改动、管理项目时使用。
- 核心价值：把“提需求、跑任务、看结果、继续迭代”放在一个闭环中，降低切换成本。

### 功能：团队管理员后台

- 功能定义：面向团队治理的后台，集中管理成员、分组、模型、镜像、宿主机与审计日志。
- 使用时机：团队初始化、权限治理、资源治理、合规审计时使用。
- 核心价值：把平台资源和成员权限统一管理，确保稳定、可控、可审计。

---

## 2. 站点与公共入口功能

### 功能：首页产品展示

- 功能定义：展示平台价值、能力亮点、客户端下载、任务模式与 Git 机器人信息。
- 使用时机：新用户首次访问、团队对外演示、产品介绍场景。
- 核心价值：降低认知门槛，快速完成“了解产品 -> 登录/注册 -> 进入控制台”的转化。

### 功能：全局导航（产品介绍/广场/使用文档）

- 功能定义：从任何时刻快速跳转到核心内容区与外部文档。
- 使用时机：用户需要在“做任务”和“查资料”之间切换时使用。
- 核心价值：减少迷路和回退操作，提升导航效率。

### 功能：用户协议与隐私政策

- 功能定义：独立展示协议与隐私内容。
- 使用时机：注册前、法务合规核验、对外采购评审时。
- 核心价值：满足合规透明要求，降低法务风险。

## 3. 账号与登录相关功能

### 功能：双入口登录（普通用户/团队管理员）

- 功能定义：一个登录页内切换两类身份登录流程。
- 使用时机：个人开发与团队治理角色切换时。
- 核心价值：减少入口分裂，统一体验但保持权限隔离。

### 功能：验证码校验登录

- 功能定义：登录前需要验证码挑战。
- 使用时机：每次账号密码登录时。
- 核心价值：降低批量撞库和滥用风险，提高账号安全性。

### 功能：记住最近登录信息

- 功能定义：本地缓存最近输入的邮箱与密码。
- 使用时机：高频登录环境（测试、演示、多次切换账号）。
- 核心价值：减少重复输入，提高登录效率。

### 功能：找回密码

- 功能定义：提交邮箱后发送重置邮件。
- 使用时机：用户忘记密码或怀疑账号泄露需要重置时。
- 核心价值：降低账号恢复成本，减少人工支持。

### 功能：重置密码

- 功能定义：携带 token 设置新密码，带成功确认。
- 使用时机：收到找回邮件后完成密码更新。
- 核心价值：闭环账号恢复流程，提升可用性与安全性。

### 功能：百智云账号登录/注册入口

- 功能定义：通过后端统一认证入口完成外部账号登录或注册。
- 使用时机：已有百智云账号的用户进入平台时。
- 核心价值：减少新账号创建阻力，提高转化。

## 4. 普通用户控制台：框架级功能

### 功能：侧边栏项目导航

- 功能定义：以“默认任务池 + 项目树 + 项目内任务”展示工作对象。
- 使用时机：用户要在多个项目/任务之间切换时。
- 核心价值：把任务组织成结构化视图，便于跟踪上下文。

### 功能：顶部面包屑

- 功能定义：展示当前位置层级（例如任务详情、项目详情）。
- 使用时机：深层页面下的定位和返回导航。
- 核心价值：降低复杂页面中的路径迷失问题。

### 功能：技术交流群入口

- 功能定义：悬浮卡展示微信群/飞书群/钉钉群二维码。
- 使用时机：用户遇到问题需要即时交流时。
- 核心价值：把支持渠道前置，减少流失和等待。

### 功能：全局刷新

- 功能定义：一键刷新当前页面数据与界面。
- 使用时机：状态可能过期、长会话后数据需重同步时。
- 核心价值：给用户可控恢复手段，降低“卡住”感。

### 功能：配置中心入口

- 功能定义：聚合 Git 身份、模型、镜像、宿主机、VM、通知等配置。
- 使用时机：任务运行前准备资源，或运行中排障时。
- 核心价值：把“任务依赖资源”集中管理，避免分散配置。

## 5. 新任务页：完整功能说明

### 功能：任务描述输入

- 功能定义：通过自然语言描述要完成的目标。
- 使用时机：发起新任务时。
- 核心价值：把技术执行入口变成需求入口，降低使用门槛。

### 功能：任务类型选择（开发/设计/审查）

- 功能定义：指定任务性质，驱动不同执行策略。
- 使用时机：明确希望产出代码、文档方案或审查意见时。
- 核心价值：提升任务意图清晰度，减少模型误判方向。

### 功能：仓库选择（我的仓库）

- 功能定义：从已授权身份拉取并选择仓库。
- 使用时机：在受管代码仓库上执行任务时。
- 核心价值：减少手工输入错误，保障权限正确性。

### 功能：仓库输入（其他仓库）

- 功能定义：手动输入仓库地址并进行格式校验。
- 使用时机：临时仓库、非列表仓库或外部仓库场景。
- 核心价值：兼容更多仓库来源，提升灵活性。

### 功能：身份凭证自动匹配

- 功能定义：根据仓库地址自动匹配可用 Git 身份。
- 使用时机：切换仓库后。
- 核心价值：降低配置复杂度，减少“仓库可见但无法操作”的问题。

### 功能：技能选择（Skills）

- 功能定义：为任务附加可用技能集合。
- 使用时机：任务需要特定流程能力时（如部署、文档化等）。
- 核心价值：增强任务执行专业性和标准化程度。

### 功能：模型选择

- 功能定义：选择执行该任务的大模型，展示可用性与健康状态。
- 使用时机：任务提交前。
- 核心价值：让用户按质量/成本/权限选择合适模型。

### 功能：分支指定

- 功能定义：指定仓库分支作为任务执行目标。
- 使用时机：避免直接在默认分支上开发，或按需求切到特定分支。
- 核心价值：降低误改风险，符合团队分支规范。

### 功能：运行环境选择（宿主机/镜像/工具）

- 功能定义：配置任务运行底座与开发工具。
- 使用时机：任务对环境有依赖（语言、系统、资源）时。
- 核心价值：保证任务可执行和结果可复现。

### 功能：语音输入

- 功能定义：通过语音转文本填充任务描述。
- 使用时机：移动办公、快速记录想法时。
- 核心价值：提升输入效率，降低表达门槛。

### 功能：文件上传（压缩包）

- 功能定义：上传附件作为任务上下文。
- 使用时机：需求涉及已有文件、规范、素材时。
- 核心价值：补充上下文，减少沟通往返。

### 功能：任务并发上限提示

- 功能定义：达到并发限制时弹窗提示。
- 使用时机：短时间内集中启动大量任务时。
- 核心价值：避免用户误以为系统异常，明确资源约束。

### 功能：任务列表与卡片

- 功能定义：展示任务摘要、状态、仓库、模型、token 等信息。
- 使用时机：任务提交后跟踪进度时。
- 核心价值：提供执行可见性，便于优先级管理。

### 功能：终止任务

- 功能定义：对正在执行任务发起停止。
- 使用时机：任务方向错误、资源占用过高或需紧急中断时。
- 核心价值：降低资源浪费，提升可控性。

### 功能：删除任务

- 功能定义：移除不再需要的任务记录。
- 使用时机：误建任务、无价值结果清理时。
- 核心价值：保持列表整洁，减少干扰。

## 6. 任务详情页：完整功能说明

### 功能：实时消息流

- 功能定义：展示模型输出、工具调用、状态变更、系统提示等。
- 使用时机：任务执行全过程。
- 核心价值：让“AI 在做什么”可解释、可追踪。

### 功能：多轮追问

- 功能定义：在同一任务上下文继续追加要求。
- 使用时机：第一次结果不完整或需增量修改时。
- 核心价值：把一次性问答升级为连续协作。

### 功能：Ask User Question 回答

- 功能定义：当系统反问时，用户可提交选项或自定义回答。
- 使用时机：任务执行遇到歧义、需要决策点时。
- 核心价值：减少误执行，提高结果贴合度。

### 功能：消息历史回放

- 功能定义：按游标加载历史轮次消息。
- 使用时机：长任务或恢复上下文时。
- 核心价值：避免上下文丢失，便于复盘。

### 功能：文件变更浏览

- 功能定义：按路径查看改动文件、差异内容与状态。
- 使用时机：验收代码结果、定位改动范围时。
- 核心价值：把“结果是否可接受”快速可视化。

### 功能：终端面板

- 功能定义：在任务对应环境执行命令并查看输出。
- 使用时机：需要人工验证、补充操作、诊断错误时。
- 核心价值：AI 自动化与人工控制无缝切换。

### 功能：预览面板

- 功能定义：展示可预览端口并打开在线预览。
- 使用时机：任务包含 Web/服务端运行结果时。
- 核心价值：缩短“改完代码 -> 验证效果”的路径。

### 功能：开发环境未就绪提示

- 功能定义：在文件、终端、预览不可用时给出明确提示。
- 使用时机：环境启动中、休眠或异常时。
- 核心价值：减少误操作与困惑。

## 7. 项目页：完整功能说明

### 功能：项目基础信息卡

- 功能定义：显示项目名称、描述、仓库绑定状态。
- 使用时机：进入项目页时。
- 核心价值：快速确认项目上下文。

### 功能：启动 AI（项目内）

- 功能定义：在项目维度直接发起开发任务（可选分支、模型、描述）。
- 使用时机：针对当前项目持续开发时。
- 核心价值：减少从项目跳回任务页的中断。

### 功能：项目自动 Review 开关

- 功能定义：为项目启用/关闭自动 Review（Webhook 驱动）。
- 使用时机：需要持续审查 PR/MR 的仓库。
- 核心价值：把代码审查从人工触发变成事件驱动。

### 功能：修改项目名称

- 功能定义：更新项目展示名称。
- 使用时机：项目命名调整或规范化。
- 核心价值：提升可读性和团队协作一致性。

### 功能：项目环境变量

- 功能定义：以键值对维护项目级环境变量。
- 使用时机：任务执行依赖 API Key、配置项时。
- 核心价值：统一配置入口，减少散落配置错误。

### 功能：项目开发镜像绑定

- 功能定义：为项目选择默认镜像。
- 使用时机：项目依赖特定运行环境时。
- 核心价值：提高任务执行稳定性和一致性。

### 功能：项目删除

- 功能定义：移除项目并自动导航到剩余项目或任务页。
- 使用时机：项目废弃或误建时。
- 核心价值：保持项目空间整洁。

### 功能：信息标签（README + 文件树）

- 功能定义：读取仓库 README 并提供项目文件浏览入口。
- 使用时机：了解项目背景、快速查看代码结构时。
- 核心价值：把“认知项目”和“动手任务”接在一起。

### 功能：需求标签（需求管理）

- 功能定义：管理项目需求，支持筛选、创建、查看、编辑。
- 使用时机：要把想法转成可执行需求时。
- 核心价值：构建“需求 -> 任务”的结构化管理链路。

### 功能：需求详情编辑

- 功能定义：编辑标题、原始需求文档、设计文档，调整状态与优先级。
- 使用时机：需求澄清、方案迭代、推进状态时。
- 核心价值：让需求文档成为活文档，不是一次性记录。

### 功能：从需求启动设计任务

- 功能定义：基于需求文本自动发起设计类任务。
- 使用时机：需求已定但设计方案未产出时。
- 核心价值：加速从需求到技术方案的转换。

### 功能：从需求启动开发任务

- 功能定义：结合需求与设计文档自动发起开发任务。
- 使用时机：方案明确后进入实现阶段。
- 核心价值：减少手工拼接上下文，提升执行一致性。

### 功能：任务标签（项目任务）

- 功能定义：查看该项目所有任务并支持删除。
- 使用时机：项目维度跟踪执行情况时。
- 核心价值：把任务从全局列表沉淀到项目上下文。

- 使用时机：查找案例、灵感和实践参考时。

## 16. 功能操作方式（可直接执行）

本节补充“怎么点、怎么配、怎么提交”的操作步骤，按页面分组。

## 16.1 登录与账号

### 操作：普通用户登录

1. 打开 ` /login `，切到“普通用户”。
2. 输入邮箱和密码，按需点击眼睛图标查看密码。
3. 点击“登录”，完成验证码。
4. 登录成功后自动进入 ` /console/tasks `。

### 操作：团队管理员登录

1. 打开 ` /login `，切到“团队管理员”。
2. 输入管理员邮箱和密码，点击“登录”。
3. 完成验证码后进入 ` /manager/dashboard `。

### 操作：找回与重置密码

1. 在登录页点击“找回密码”进入 ` /findpassword `。
2. 输入邮箱并提交，完成验证码。
3. 打开邮件中的重置链接进入 ` /resetpassword?token=... `。
4. 输入新密码并确认，提交完成重置。

## 16.2 新任务页

### 操作：发起一个开发任务

1. 进入 ` /console/tasks `。
2. 在输入框填写需求，如“修复登录页验证码报错”。
3. 任务类型选择“开发”。
4. 选择仓库来源（我的仓库/其他仓库），并确认仓库地址。
5. 选择分支、模型、镜像、宿主机、开发工具。
6. 需要时添加技能、上传附件或使用语音输入。
7. 点击提交，成功后自动跳转任务详情。

### 操作：查看并管理任务列表

1. 保持在 ` /console/tasks ` 页面下半区。
2. 点击任务标题进入详情。
3. 点击任务卡片右侧菜单，可执行“终止任务”或“删除任务”。
4. 向下滚动触发自动加载更多任务。

## 16.3 任务详情页

### 操作：在已有任务中继续追问

1. 打开 ` /console/task/:taskId `。
2. 在底部输入框补充要求，如“再补一个单元测试”。
3. 点击发送，等待新一轮执行输出。

### 操作：回答系统提问（Ask User Question）

1. 当消息区出现提问卡片时，先阅读问题和选项。
2. 选择推荐项或填写自定义答案。
3. 提交后继续观察后续执行消息。

### 操作：查看代码改动

1. 在详情页打开文件/改动面板。
2. 点击文件名查看变更内容。
3. 需要复核时切换到其他改动文件逐项检查。

### 操作：使用任务终端与预览

1. 在详情页打开终端面板执行验证命令。
2. 打开预览面板查看可用端口。
3. 点击端口链接打开页面验收效果。

## 16.4 项目页

### 操作：创建并维护需求

1. 进入 ` /console/project/:projectId `，切到“需求”标签。
2. 点击“创建需求”，填写标题与原始需求说明并提交。
3. 点击某条需求打开详情。
4. 在详情中可编辑标题、原始需求、设计文档。
5. 通过右侧下拉选择状态和优先级并保存。

### 操作：从需求启动设计任务

1. 在需求详情点击右上菜单。
2. 选择“启动设计任务”。
3. 在弹窗中确认项目、仓库分支。
4. 点击“开始设计”，系统跳转到新任务详情。

### 操作：从需求启动开发任务

1. 在需求详情点击右上菜单。
2. 选择“启动开发任务”。
3. 在弹窗中确认分支后点击“开始开发”。
4. 系统创建开发任务并跳转详情页。

### 操作：项目级启动 AI

1. 在项目信息卡点击“启动 AI”。
2. 填写任务描述并选择模型、分支。
3. 点击提交后跳转 ` /console/task/:taskId `。

### 操作：项目自动 Review

1. 在项目信息卡点击“未开启自动 Review/已开启自动 Review”按钮。
2. 在弹窗中切换开关。
3. 成功后返回项目页，按钮状态同步更新。

### 操作：配置项目环境变量与镜像

1. 在项目右上菜单选择“环境变量”或“开发镜像”。
2. 环境变量：逐行填 Key/Value，点击保存。
3. 开发镜像：选择镜像后点击保存。

## 16.5 Git 机器人

### 操作：创建机器人并配置回调

1. 进入 ` /console/gitbot `，切到“配置”。
2. 点击新增机器人，填写平台与名称等信息。
3. 创建后在机器人菜单里点“WebHook”。
4. 复制 WebHook URL 和 Secret Token。
5. 到代码平台仓库中配置 Webhook 回调。

### 操作：查看机器人审查任务

1. 在 ` /console/gitbot ` 切到“任务”。
2. 查看任务状态（等待/执行中/完成/失败）。
3. 点击任务标题打开关联 PR 查看结果。

## 16.6 设置中心

### 操作：绑定 Git 身份

1. 左下点击“配置”，进入“Git 身份”。
2. 选择平台（GitHub/Gitee/Gitea/GitLab）并点击绑定。
3. 在弹出的授权页完成授权，返回后自动刷新列表。

### 操作：新增并验证模型

1. 切到“AI 大模型”，点击新增。
2. 填写模型参数并保存。
3. 在模型项菜单点击“检查”。
4. 检查通过后可设为默认模型。

### 操作：新增镜像与设默认

1. 切到“系统镜像”，点击新增并保存镜像。
2. 在镜像菜单点击“设为默认”。
3. 需要维护时可编辑或移除。

### 操作：绑定宿主机

1. 切到“宿主机”，点击“绑定宿主机/安装命令”。
2. 复制安装命令到目标机器执行。
3. 回到页面刷新，确认宿主机出现并在线。
4. 在菜单中可设默认、改备注、移除。

### 操作：创建开发环境 VM

1. 切到“开发环境”，点击“创建开发环境”。
2. 选择宿主机与镜像并提交。
3. 待状态变为可用后，可点击“终端”或“文件”。
4. 需要延长使用时点击“续期”。

### 操作：配置通知渠道

1. 切到“通知”，点击“新增渠道”。
2. 选择渠道类型，填写名称、Webhook URL、Secret（可选）。
3. 勾选至少一种订阅事件，保存。
4. 在渠道菜单点击“测试”验证推送。

## 16.7 终端、文件、远程协助

### 操作：在终端页连接环境

1. 打开 ` /console/terminal?envid=... `。
2. 若有多终端连接，选择目标连接。
3. 等待连接成功后执行命令。

### 操作：文件管理常用操作

1. 打开 ` /console/files?envid=...&path=... `。
2. 使用顶部按钮新建文件/目录或上传文件。
3. 点击文件可在线编辑，保存后生效。
4. 在文件菜单中可下载、复制、移动、删除。

### 操作：发起远程协助

1. 在终端页点击“远程协助”。
2. 选择协助模式并生成密码。
3. 复制分享链接与密码给协作者。
4. 协作者打开 ` /sharedterminal?id=... ` 输入密码接入。

## 16.8 开发者广场

### 操作：发布文章

1. 进入 ` /playground/create `。
2. 选择“文章”，填写标题与正文。
3. 按需上传图片后提交。

### 操作：发布任务分享

1. 在 ` /playground/create ` 选择“任务分享”。
2. 关联一个历史任务。
3. 选择需要分享的代码文件并打包上传。
4. 提交后等待审核展示。

### 操作：查看分享详情与代码

2. 阅读图文内容。
3. 在代码区域展开目录查看文件。
4. 如有任务回放，加载消息查看完整过程。

## 16.9 管理员后台

## 17. 标准操作卡（前置条件/步骤/预期/失败处理）

本节用于培训和测试执行，按“可直接验收”的格式给出关键高频功能。

## 17.1 用户登录

- 前置条件：已有普通用户账号；网络可访问登录服务。
- 操作步骤：打开 ` /login ` -> 选择“普通用户” -> 输入邮箱密码 -> 点击登录并完成验证码。
- 预期结果：跳转到 ` /console/tasks `，页面显示任务输入区和任务列表。
- 常见失败与处理：
  - 验证码失败：重新发起验证码挑战后再提交。
  - 密码错误：使用“找回密码”进入 ` /findpassword ` 重置。
  - 登录后仍在登录页：检查浏览器是否禁用 cookie/localStorage。

## 17.2 创建开发任务

- 前置条件：至少绑定 1 个 Git 身份；至少有 1 个可用模型。
- 操作步骤：进入 ` /console/tasks ` -> 填任务内容 -> 选“开发” -> 选仓库/分支/模型/镜像/宿主机 -> 提交。
- 预期结果：提示任务创建成功并自动跳转 ` /console/task/:taskId `。
- 常见失败与处理：
  - 未找到模型：到“配置 -> AI 大模型”新增并健康检查。
  - 仓库无权限：到“配置 -> Git 身份”重新授权对应平台。
  - 触发并发上限：关闭或等待其他任务结束后重试。

## 17.3 任务详情追问与验收

- 前置条件：任务已成功创建并可打开详情页。
- 操作步骤：在 ` /console/task/:taskId ` 输入追问 -> 发送 -> 打开文件改动面板复核 -> 打开预览端口验收。
- 预期结果：消息流新增一轮执行记录；可看到新增改动；可打开预览页面。
- 常见失败与处理：
  - 无法追问：确认任务未被终止且输入框可用。
  - 改动面板为空：等待任务执行完成，或查看是否仅返回建议未落地修改。
  - 无预览端口：在终端执行启动命令，确认服务已监听端口。

## 17.4 项目创建需求并启动设计任务

- 前置条件：已进入某项目页 ` /console/project/:projectId `。
- 操作步骤：切换到“需求” -> 创建需求（标题 + 原始需求）-> 打开需求详情 -> 菜单点“启动设计任务” -> 选分支并提交。
- 预期结果：创建设计任务并跳转到对应任务详情。
- 常见失败与处理：
  - 分支加载失败：检查仓库绑定与网络，再手工输入分支。
  - 提交无响应：刷新页面后重试，并确认需求标题非空。

## 17.5 项目启动开发任务

- 前置条件：项目已绑定仓库；模型可用。
- 操作步骤：在项目信息卡点击“启动 AI” -> 输入任务说明 -> 选分支和模型 -> 提交。
- 预期结果：系统创建开发任务并跳转任务详情。
- 常见失败与处理：
  - 提示请选择分支：非 internal 仓库必须指定分支。
  - 模型不可用：切换到可用模型，或先做模型健康检查。

## 17.6 配置 Git 机器人并接入仓库 Webhook

- 前置条件：用户已能访问目标代码仓库的 Webhook 配置页。
- 操作步骤：进入 ` /console/gitbot ` -> 新建机器人 -> 打开 WebHook 弹窗 -> 复制 URL 和 Secret -> 到代码平台配置回调并保存。
- 预期结果：仓库事件触发后，机器人任务页出现新的审查任务记录。
- 常见失败与处理：
  - 没有任务产生：检查仓库事件类型是否勾选 PR/MR。
  - 回调失败：检查 URL 可达性与 Secret 是否一致。

## 17.7 绑定宿主机并创建 VM

- 前置条件：有可执行安装命令的目标机器；机器网络可连平台。
- 操作步骤：配置页进入“宿主机” -> 复制安装命令并在机器执行 -> 回页面刷新确认宿主机在线 -> 进入“开发环境”创建 VM。
- 预期结果：宿主机出现在列表并可设默认；VM 状态由 pending 变为可用。
- 常见失败与处理：
  - 宿主机不出现：确认安装命令执行成功并检查机器网络。
  - VM 长时间 pending：检查宿主机资源是否不足或镜像拉取失败。

## 17.8 配置通知渠道并发送测试消息

- 前置条件：已获取钉钉/飞书/企业微信/Webhook 的可用地址。
- 操作步骤：进入“配置 -> 通知” -> 新增渠道 -> 填名称和 Webhook URL -> 勾选订阅事件 -> 保存 -> 点击“测试”。
- 预期结果：配置保存成功，目标渠道收到测试消息。
- 常见失败与处理：
  - 保存失败：检查是否至少勾选一个事件。
  - 测试失败：检查 Webhook URL 或 Secret 是否正确。

## 17.9 终端远程协助

- 前置条件：已有可用 VM，且当前用户能打开终端页。
- 操作步骤：打开 ` /console/terminal?envid=... ` -> 点击“远程协助” -> 选择模式并生成密码 -> 分享链接和密码 -> 协作者在 ` /sharedterminal?id=... ` 输入密码接入。
- 预期结果：协作者成功进入共享终端，双方可看到连接状态。
- 常见失败与处理：
  - 协作者无法连接：确认链接中的 id 与密码匹配且未过期。
  - 连接后无输出：让主会话先执行一条命令确认终端活跃。

## 17.10 管理员添加成员并分配分组

- 前置条件：当前账号为管理员；团队成员上限未满。
- 操作步骤：进入 ` /manager/members ` -> 添加成员（支持多邮箱）-> 选择分组并提交 -> 在分组卡片检查成员归属。
- 预期结果：成员列表出现新增用户，且在对应分组中可见。
- 常见失败与处理：
  - 邮箱格式报错：按逗号/换行分隔并修正非法邮箱。
  - 无法添加：检查团队成员额度是否达到上限。

## 18. 常见问题标准回答（对外口径）

本节用于覆盖“随问随答”场景。每个问题均给出可直接复述的标准回答。

### FAQ 1：专业版是什么，卖多少钱，有什么权益？

- 标准回答：专业版是面向高强度研发协作的订阅计划，通常提供更高可用模型范围、更高并发与更完整资源能力。具体价格与权益以当前“方案/版本”页面实时展示为准。
- 用户查看路径：首页 -> 方案/版本信息（或购买页）-> 对比各版本权益。
- 交付注意：
  - 报价以页面实时信息为准，避免引用过期数字。
  - 对外沟通建议同时给出“权益范围 + 生效口径 + 更新时间”。

### FAQ 2：怎么下载任务里产生的文件？

- 标准回答：先在任务详情确认生成的文件路径，再进入文件管理页下载对应文件。
- 操作路径：任务详情 ` /console/task/:taskId ` -> 文件/改动面板定位文件 -> 文件管理 ` /console/files?envid=...&path=... ` -> 文件菜单点击“下载”。
- 适用场景：需要把 AI 产物发给同事、归档、二次加工时。

### FAQ 3：Git 身份是什么？怎么用？有什么用？

- 标准回答：Git 身份是平台与代码托管平台（GitHub/Gitee/Gitea/GitLab）之间的授权凭证，用来访问仓库、读取分支并在有权限的仓库中执行任务。
- 操作路径：配置 -> Git 身份 -> 选择平台 -> 完成授权 -> 返回后自动刷新。
- 业务价值：
  - 让仓库选择与分支拉取可用。
  - 降低权限不匹配导致的任务失败。
  - 支持基于仓库上下文的自动化开发与审查。

### FAQ 4：任务并发上限是什么意思？

- 标准回答：并发上限表示同一账号/团队同时运行任务数量的限制，超限时会弹出限制提示，需等待部分任务结束或升级版本后继续。
- 操作建议：先终止不必要任务，再重试提交。

### FAQ 5：为什么我看不到可用模型？

- 标准回答：通常是模型未配置、健康检查失败或订阅权限不满足。
- 排查路径：配置 -> AI 大模型 -> 新增/检查模型 -> 确认可用后再创建任务。

### FAQ 6：怎么把任务结果分享给别人？

- 标准回答：可通过两种方式分享：
  - 文件分享：在文件管理下载后外部发送。
  - 内容分享：在开发者广场发布“任务分享”，附带代码包和过程说明。
- 操作路径：` /playground/create ` -> 选择“任务分享” -> 关联历史任务 -> 选择代码文件 -> 提交。

### FAQ 7：怎么邀请别人一起看我的终端？

- 标准回答：在终端页发起远程协助，生成分享链接和密码后给协作者。
- 操作路径：` /console/terminal?envid=... ` -> 远程协助 -> 生成密码 -> 分享链接 -> 协作者在 ` /sharedterminal?id=... ` 输入密码接入。

### FAQ 8：如何保证任务在指定分支上执行？

- 标准回答：创建任务时在分支字段显式指定目标分支，项目内启动任务时也可在弹窗中选择分支。
- 操作路径：
  - 新任务页：` /console/tasks ` 填写分支。
  - 项目弹窗：` /console/project/:projectId ` -> 启动 AI -> 选择分支。

### FAQ 9：自动 Review 怎么开，开了有什么用？

- 标准回答：在项目页开启“自动 Review”后，系统会为仓库配置自动审查触发链路，PR/MR 到来时可自动发起审查任务。
- 操作路径：项目页 -> 自动 Review 按钮 -> 弹窗开关启用。
- 价值：把审查从手工触发变成事件驱动，提升代码质量门槛一致性。

### FAQ 10：管理员怎么快速完成团队初始化？

- 标准回答：先配资源，再配人，最后开通知。
- 推荐顺序：
  1. ` /manager/hosts ` 绑定宿主机
  2. ` /manager/models ` 配置模型并健康检查
  3. ` /manager/images ` 配置镜像
  4. ` /manager/members ` 添加成员与分组
  5. ` /manager/other-settings ` 配置团队通知
