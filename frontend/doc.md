# MonkeyCode AI 前端使用文档

本文档基于当前 `frontend` 源码整理，描述的是现在前端已经实现的页面、入口和交互。

如果文档与页面行为不一致，优先以以下源码为准：

- 路由入口：`frontend/src/App.tsx`
- 页面组件：`frontend/src/pages/**`
- 关键交互组件：`frontend/src/components/**`

---

## 一、产品与页面入口

### 1. 当前前端覆盖的能力

当前前端已经覆盖以下几类能力：

- 官网首页与公开介绍页
- 普通用户登录与控制台
- 智能任务创建、执行、追问与文件查看
- 项目协作与需求管理
- Git 审查机器人
- 独立开发环境、终端、文件管理、远程协助
- 开发者广场内容浏览与发布
- 企业管理后台

### 2. 主要路由

- 首页：`/`
- 登录页：`/login`
- 找回密码：`/findpassword`
- 重置密码：`/resetpassword?token=...`
- 开发者广场：`/playground`
- 发布广场内容：`/playground/create`
- 广场详情：`/playground/detail?id=...`
- 隐私政策：`/privacy-policy`
- 用户协议：`/user-agreement`
- 普通用户控制台：`/console`
- 新任务页：`/console/tasks`
- 任务详情页：`/console/task/:taskId`
- 项目页：`/console/project/:projectId`
- Git 机器人：`/console/gitbot`
- IDE 辅助工具：`/console/ide`
- 独立终端页：`/console/terminal?envid=...`
- 独立文件管理页：`/console/files?envid=...&path=...`
- 远程协助页：`/sharedterminal?id=...`
- 企业管理后台：`/manager`

### 3. 首页导航

首页顶部导航当前包含：

- `产品介绍`
- `广场`
- `使用文档`
- 已登录时显示 `控制台`
- 未登录时显示 `注册`、`登录`

说明：

- `使用文档` 当前跳转到外部文档站，不是本仓库里的 `doc.md`
- 移动端导航收纳在抽屉菜单中

### 4. 首页内容

首页当前由以下区块组成：

- Banner：免费 AI 编程平台介绍
- 能力亮点
- 全平台客户端下载
- 智能任务模式展示
- Git 机器人介绍
- 版本与方案
- 页脚资源链接、隐私政策、用户协议、技术交流群二维码

### 5. 客户端下载

首页“全平台客户端”区块当前展示：

- Windows
- macOS
- Android
- iOS

下载统一跳转：

- `https://github.com/chaitin/MonkeyCode/releases`

## 二、登录与账号

### 1. 登录页

登录页路径：

- `/login`

页面包含两个标签：

- `普通用户`
- `团队管理员`

两种登录表单都支持：

- 输入邮箱和密码
- 密码明文 / 隐藏切换
- 登录前验证码校验
- 本地记住上次输入的邮箱和密码

### 2. 普通用户登录

普通用户登录成功后进入：

- `/console/`

普通用户标签下还提供：

- `找回密码`
- `百智云账号登录`
- `注册`

说明：

- 当前前端没有单独的注册页
- “百智云账号登录”和“注册”都通过后端登录入口发起

### 3. 团队管理员登录

团队管理员登录成功后进入：

- `/manager/`

团队管理员标签下当前没有单独的“找回密码”按钮。

### 4. 找回密码

找回密码页路径：

- `/findpassword`

操作流程：

1. 输入邮箱
2. 完成验证码校验
3. 提交后发送重置密码邮件

### 5. 重置密码

重置密码页路径：

- `/resetpassword?token=...`

页面会先根据 `token` 读取账号信息，再要求输入：

- 新密码
- 确认密码

重置成功后会弹出确认框，关闭后返回首页。

## 三、普通用户控制台

### 1. 默认入口

登录后访问 `/console` 会自动跳转到：

- `/console/tasks`

控制台布局当前为：

- 左侧侧边栏
- 顶部面包屑与刷新区
- 主内容区

### 2. 左侧侧边栏

侧边栏分为三块：

1. `开发项目`
2. `配置`
3. 钱包与当前用户

#### 开发项目

“开发项目”区域包含：

- `默认`
- 用户创建的项目列表

支持的交互：

- 刷新项目与未归属任务列表
- 创建项目
- 展开 / 折叠项目
- 自动记住展开状态
- 每 30 秒自动刷新一次项目和任务列表
- 如果项目或未归属任务里存在执行中的任务，会自动保持展开

`默认` 分组对应未归属项目的任务，点击进入：

- `/console/tasks`

未归属任务支持：

- 进入任务详情
- 对执行中任务执行 `终止`
- 对所有任务执行 `删除`

项目支持：

- 进入项目概览页
- 查看项目下已有任务
- 当项目已绑定仓库时，直接 `启动 AI`

#### 创建项目

创建项目当前不是自由输入仓库地址，而是：

1. 输入项目名称
2. 从已绑定的 Git 身份中选择一个身份
3. 从该身份可访问的授权仓库列表里选择仓库

如果尚未绑定 Git 身份，弹窗会提示先去“配置”里完成绑定。

### 3. 顶部区域

顶部当前包含：

- 面包屑导航
- `技术交流群`
- `刷新`

“技术交流群”悬浮后会显示：

- 微信群二维码
- 飞书群二维码
- 钉钉群二维码

### 4. 钱包

钱包入口位于左侧侧边栏底部。入口本身会直接显示两类信息：

- 左侧显示当前套餐简称：
  - `基础版`
  - `专业版`
- 右侧显示当前可用积分总数。

这里显示的可用积分并不是单独某一类余额，而是前端把 `总积分` 和 `今日积分` 相加后展示出来，方便用户直接看到当前还能使用多少积分。

点开后，钱包弹窗分为四个分区：

- `积分余额`
- `我的套餐`
- `赚积分`
- `积分记录`

#### 4.1 积分余额

`积分余额` 页面由两块卡片组成。

第一块是 `积分概览`，会明确拆分展示：

- `总积分`：账户主积分余额
- `今日积分`：当日单独统计的积分余额

侧边栏入口展示的是两者合计值，钱包内部则会拆开显示，便于用户理解自己当前积分来源。

第二块是 `账户状态`，当前前端会展示：

- `当前身份`，显示为 `普通会员` 或 `专业会员`
- 如果当前是专业版，还会额外显示 `有效期至`
- 如果当前是专业版，还会额外显示 `自动续费` 当前状态

`积分余额` 页面下方还有 `充值积分` 区域，当前前端内置了 3 个固定充值档位：

- `10,000 积分`，价格 `¥50`
- `50,000 积分`，价格 `¥200`，原价 `¥250`，页面标记为 `8 折`
- `300,000 积分`，价格 `¥1000`，原价 `¥1500`，页面标记为 `6.7 折`

交互方式如下：

- 用户先点击某个充值档位卡片
- 再点击 `确认充值`
- 前端向后端请求支付链接
- 请求成功后会在新标签页打开支付页面

如果没有先选档位，点击确认时会提示用户先选择充值套餐。

#### 4.2 我的套餐

`我的套餐` 页面会先显示当前正在使用的套餐。如果用户已经是专业版，这里会额外显示到期日期。

页面中并排展示 `基础版` 和 `专业版` 两张说明卡，当前前端写死的权益说明如下。

基础版：

- 免费
- 官方指定免费模型
- 最多同时运行 `1` 个任务
- 不含每日赠送积分

专业版：

- 价格为 `10,000 积分 / 月`
- 可选择更多 AI 模型
- 最多同时运行 `3` 个任务
- 每日赠送 `2000` 积分

开通逻辑也需要写清楚：

- 只有当账户当前可用积分大于等于 `10,000` 时，按钮才允许开通专业版
- 当前端判断积分不足时，按钮会直接显示 `积分不足`
- 当前端判断积分足够时，按钮会显示 `开通专业版（10,000 积分）`
- 点击后会直接发起开通请求，并刷新钱包余额、套餐状态和积分记录

如果用户已经是专业版，则不会再显示开通按钮，而是显示自动续费设置卡片。这里有两个重要行为：

- 可以通过开关控制 `自动续费`
- 前端会同步显示状态说明：
  - 已开启：`已开启，套餐到期后自动续费`
  - 未开启：`未开启，到期后不会自动续费`

#### 4.3 赚积分

`赚积分` 页面现在包含两部分：`兑换积分` 和 `邀请注册`。

`兑换积分`：

- 页面提供一个兑换码输入框
- 用户输入兑换码后可点击 `兑换`
- 按回车也会直接触发兑换
- 兑换成功后会刷新钱包余额

`邀请注册` 区域提供以下能力：

- 系统会自动生成个人邀请链接，格式为：
  - `https://monkeycode-ai.com/?ic=当前用户ID`
- 点击 `复制邀请链接` 会把完整链接复制到剪贴板
- 页面明确提示：好友通过该链接完成注册后，邀请人可获得 `5000 积分`
- 奖励提示角标也会直接显示 `+5,000`

邀请记录区会展示：

- 已邀请总人数
- 最近邀请用户列表

每条邀请记录会显示：

- 用户头像
- 用户昵称；如果没有昵称则显示 `未命名用户`
- 注册时间的相对描述，例如“几分钟前注册”
- 本次邀请带来的积分奖励，右侧以 `+积分数` 的形式显示

当前前端首次会请求前 `20` 条邀请记录，页面展示最近一批邀请用户明细。

#### 4.4 积分记录

`积分记录` 页面用于查看积分收支流水，列表支持滚动加载更多，当前单次按页请求 `10` 条记录。

页面会把积分变动按正负显示，常见记录类型包括：

- `新用户注册奖励`
- `通过兑换码领取`
- `邀请注册奖励`
- `开发环境消耗`
- `大模型消耗`
- `兑换专业版`
- `专业版自动续费`
- `当日钱包发放`
- `充值积分`

对用户来说，这一页最重要的作用是分清三类消耗或收入来源：

- 哪些积分来自注册奖励、邀请奖励、兑换码或每日发放
- 哪些积分消耗在开发环境或模型调用
- 哪些积分用于开通专业版或专业版自动续费

### 5. 用户菜单

侧边栏最下方的用户菜单支持：

- 修改头像
- 修改昵称
- 绑定邮箱
- 修改密码 / 设置密码
- 登出

补充说明：

- 头像文件必须是图片，且不超过 5MB
- 已绑定邮箱时，“绑定邮箱”按钮会禁用
- 已有密码的账号修改密码时需要输入当前密码

## 四、智能任务

### 1. 新任务页

新任务页路径：

- `/console/tasks`

页面由两部分组成：

- 上方任务输入区
- 下方任务卡片列表

### 2. 创建任务

#### 任务输入

主输入框支持：

- 输入自然语言需求
- `Shift + Enter` 换行
- 语音输入

#### 任务类型

当前有三种类型：

- `开发`
- `设计`
- `审查`

说明：

- `开发`：执行编码任务
- `设计`：输出技术方案与设计文档
- `审查`：审查代码并识别风险

#### 代码来源

“代码”菜单当前支持三种来源：

- `ZIP 文件`
- `我的仓库`
- `其他仓库`

`ZIP 文件`：

- 仅接受 `.zip`
- 文件大小上限 10MB
- 先上传，再作为本次任务的代码来源

`我的仓库`：

- 按已绑定 Git 身份分组展示
- 仅展示当前身份下拉到的授权仓库
- 支持在每个身份下搜索仓库
- 选中后自动关联对应 Git 身份凭证

`其他仓库`：

- 手动输入完整仓库地址
- 回车确认
- 下方会展示从任务列表里归纳出的历史仓库地址

#### 技能

“技能”按钮会打开技能选择面板，当前实现为：

- 按标签分类展示
- 支持多选
- 部分默认技能不可取消
- 按钮上显示已选技能数量

#### 执行前参数确认

点击 `执行` 后会打开“任务参数”弹窗。

当前弹窗字段包括：

- 大模型
- 仓库分支
- 仓库身份凭证

仅子账号额外显示：

- 宿主机
- 系统镜像

补充说明：

- 从“我的仓库”里选中的仓库，不再重复要求选择身份凭证
- 使用 ZIP 代码源时，不显示仓库分支和仓库身份凭证
- 当前创建的任务固定使用 `OpenCode`
- 如果选择了平台内置模型，会强制使用平台内置宿主机
- 任务创建时固定资源为 2 核 CPU、8GB 内存、3 小时生命周期

#### 并发限制

如果达到任务并发上限，会弹出并发限制提示框。

弹窗会：

- 列出当前正在运行或等待中的任务
- 允许直接终止这些任务
- 对基础版用户提供升级专业版入口

### 3. 任务列表

任务列表以卡片方式展示，当前每页加载 24 条，支持无限滚动。

卡片信息包括：

- 任务标题
- 仓库名或 ZIP 文件名
- 任务状态
- Token 用量
- 创建时间

任务状态当前对应：

- `等待执行`
- `正在执行`
- `任务完成`
- `执行失败`

卡片支持：

- 点击标题进入任务详情
- 右上角菜单对执行中任务执行 `终止`
- 右上角菜单对所有任务执行 `删除`
- 悬浮查看任务内容、类型、模型、分支、开发工具等详细信息

## 五、任务详情页

### 1. 路径

- `/console/task/:taskId`

### 2. 页面结构

当前任务详情页主要由以下区域组成：

- 顶部状态栏
- 中央消息区
- 右侧文件面板
- 底部终端面板
- 预览弹窗

顶部按钮当前只有：

- `终端`
- `文件`
- `预览`

说明：

- `文件` 按钮会显示当前已变更文件数量
- `预览` 按钮会显示当前发现的可预览端口数量
- 当开发环境不可用时，这三个按钮会禁用

顶部状态栏还会显示：

- 当前任务使用的大模型
- 输入 / 输出 token 统计

### 3. 环境准备状态

如果任务对应开发环境尚未就绪，主区域会显示准备视图。

当前会展示：

- 开发环境状态说明
- 最近一次环境条件消息

这部分不仅覆盖“正在创建中”，也覆盖部分环境失败状态。

### 4. 消息区

消息区用于展示任务的完整对话和工具执行信息，支持：

- 自动附着正在执行中的实时消息流
- 加载历史轮次消息
- 自动滚动到底部
- 一键滚动到顶部 / 底部

如果任务已经结束、但当前没有实时流消息，页面会自动去加载历史轮次。

### 5. 与 AI 对话

输入区支持：

- `Enter` 发送
- `Shift + Enter` 换行
- 语音输入
- 斜杠命令插入

输入框左侧菜单当前包含：

- `重置上下文`
- `重新加载开发工具`

当前实现里有一个容易和旧文档混淆的点：

- 只要开发环境仍在线，任务结束后仍然可以继续向当前任务追加新一轮对话
- 只有在开发环境不可用时，底部才显示 `任务已结束`

任务执行中时：

- 文本输入框会隐藏
- 发送按钮切换为 `停止`
- 页面显示当前耗时

### 6. AI 反向提问

任务消息流里当前支持 `ask_user_question` 类型消息。

也就是说，AI 在执行过程中可以直接在消息区向用户发起补充提问，当前实现支持：

- 单选题
- 多选题
- 自定义补充答案
- 直接在消息卡片中提交回答

### 7. 文件面板

点击顶部 `文件` 后，会在右侧展开文件面板。

文件面板当前包含两个模式：

- `目录`
- `变更`

#### 目录模式

支持：

- 浏览任务工作区文件树
- 展开 / 折叠目录
- 打开文本文件只读预览
- 刷新文件树
- 在当前目录创建文件
- 创建目录
- 上传文件
- 复制文件 / 目录
- 移动文件 / 目录
- 下载文件 / 目录
- 删除文件 / 目录

补充说明：

- 文件预览为只读
- 超过 100KB 的文件不支持预览
- 二进制文件不支持预览

#### 变更模式

支持：

- 查看全部已变更文件列表
- 显示新增 / 删除行数
- 点击单个文件查看 Diff
- Diff 以分栏形式展示

说明：

- 旧版文档里的独立“修改”面板已经不存在
- 变更查看能力已经合并进右侧文件面板的 `变更` 模式

### 8. 终端面板

点击顶部 `终端` 后，会在页面下方展开终端区。

终端区当前支持：

- 查看当前任务绑定开发环境的终端会话列表
- 新建终端会话
- 切换终端会话
- 刷新会话列表

## 十四、与旧文档相比最容易出错的点

这里列出当前最容易和旧版说明冲突的地方，方便后续校对。

- 首页的“使用文档”现在跳转外部文档站，不是前端内页
- 钱包入口在左侧侧边栏底部，不在顶栏
- 创建项目时现在必须从已绑定身份的授权仓库里选仓库
- 任务详情页现在是顶部 `终端 / 文件 / 预览` 按钮，不再是旧版底部工具条
- 任务详情页没有独立“修改”面板，变更查看已合并到文件面板的 `变更` 模式
- 任务详情页支持 AI 在消息流里向用户发起补充提问
- 只要开发环境仍在线，任务结束后仍可继续在任务详情页追加新一轮对话
- 任务详情文件预览只读，且大于 100KB 或二进制文件不支持预览
- 项目信息页现在是只读仓库浏览器 + README 渲染，不是开发环境文件管理页
- 独立文件管理页当前不能直接删除目录，只能删除文件
- Git 机器人页面仍没有固定侧边栏入口
- 广场详情里的评论区仍是占位
- IDE 辅助工具页仍是占位页
- 企业管理后台仪表盘仍是占位页
- `/tasks/public` 只是跳转页，不是独立公开列表页
